# Seele v0.5 — PRD (Product Requirements Document)

> 版本：v0.5
> 状态：Draft
> 日期：2026-07-07

## P0 — 核心架构重构

### PRD-001: Agent 层提升与解耦

**需求**：`core/agent/` 提升为顶级 `agent/`，承担编排层职责。

**验收标准**：
- [ ] `agent/` 目录包含 `api/`、`tool/`、`gateway/` 子包
- [ ] `agent.New()` 接口签名不变（`Options` → `(*Agent, error)`）
- [ ] `agent.Agent` 不直接持有 `session.Holder` 引用（通过 `NewSession()` 返回）
- [ ] 编译通过，所有测试通过

**优先级**：P0

### PRD-002: 双网关分离

**需求**：将"选号"和"工具路由"拆为两个独立 Gateway。

**验收标准**：
- [ ] `agent/gateway/api.Gateway` 接口定义：`Select()`, `Health()`, `Register()`
- [ ] `agent/gateway/tool.Gateway` 接口定义：`VisibleTools()`, `Dispatch()`, `ActivatePlugin()`, `DeactivatePlugin()`
- [ ] 两个 Gateway 相互独立，无直接依赖
- [ ] 默认实现覆盖核心场景：API Gateway 用 round-robin，Tool Gateway 用 plugin-based

**优先级**：P0

### PRD-003: LLM Client 桥接双网关

**需求**：LLM Client 位于双网关之间，管理 ReAct loop。

**验收标准**：
- [ ] `agent/api.LLMClient` 同时持有 `apigw.Gateway` 和 `toolgw.Gateway`
- [ ] `Chat()` 方法内部完成：选号 → 获取工具列表 → LLM 调用 → 工具调度 → 循环
- [ ] 默认 maxLoops = 8
- [ ] 流式 `ChatStream()` 支持

**优先级**：P0

### PRD-004: Session 提升为 Context

**需求**：`core/session/` 提升为顶级 `context/`，合并 `history/`。

**验收标准**：
- [ ] `context/` 目录包含 `holder.go`、`chat.go`、`dispatch.go`、`history.go`、`compress.go`、`limit.go`、`storage.go`、`config.go`
- [ ] `context.Holder` 接口：`Chat()`, `ChatStream()`, `History()`, `SessionID()`
- [ ] 上下文压缩/限制功能从 `history/` 合并到 `context/`
- [ ] `history/` 目录删除

**优先级**：P0

### PRD-005: Context 本地存储

**需求**：`context/` 提供默认的本地存储实现。

**验收标准**：
- [ ] `LocalStorage` 接口：`Save(key, data)`, `Load(key)`, `List(prefix)`
- [ ] `FileStorage` 实现：基于文件系统的持久化
- [ ] 会话数据可持久化到本地磁盘
- [ ] 存储路径可配置

**优先级**：P1

## P1 — 图编排原语

### PRD-006: Graph 抽象原语

**需求**：`workplan/` 简化为纯抽象 `graph/` 包。

**验收标准**：
- [ ] `graph.Node` 接口：`Name()`, `Process(ctx, input) (output, error)`
- [ ] `graph.Edge` 类型：支持 Direct / Conditional / Parallel 三种
- [ ] `graph.Graph` 结构：`AddNode()`, `AddEdge()`, `Compile() → Runnable`
- [ ] `graph.Runner` 执行引擎：`Run(ctx, input) (output, error)`
- [ ] `workplan/` 目录删除或改为包装 `graph/`

**优先级**：P1

## P2 — 外部 Provider 对齐

### PRD-007: Provider import 更新

**需求**：`provider/` 下的实现更新 import 路径到 `agent/tool`。

**验收标准**：
- [ ] `provider/hub_provider.go` import `agent/tool` 而非 `core/tool`
- [ ] `provider/mcp_provider.go` import `agent/tool`
- [ ] `provider/inline_provider.go` import `agent/tool`
- [ ] `provider/tool_provider.go` type alias 指向 `agent/tool`

**优先级**：P2

## P3 — 示例与测试更新

### PRD-008: 示例更新

**需求**：`example_Implement/` 下的示例使用新架构。

**验收标准**：
- [ ] 所有示例编译通过
- [ ] 示例展示双网关、Plugin 切换、Context 存储等新特性

**优先级**：P3

### PRD-009: 测试覆盖

**需求**：核心模块的新增代码有单元测试。

**验收标准**：
- [ ] `agent/gateway/api/` 测试：Select 轮询、故障转移
- [ ] `agent/gateway/tool/` 测试：Plugin 过滤、Dispatch
- [ ] `context/storage.go` 测试：Save/Load/List
- [ ] `graph/` 测试：Graph 构建与执行

**优先级**：P3
